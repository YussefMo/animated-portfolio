## Security Audit (Before)

### Current Headers

| Header | Status |
|---|---|
| Content-Security-Policy | ❌ Missing |
| Strict-Transport-Security | ❌ Missing |
| X-Frame-Options | ❌ Missing |
| X-Content-Type-Options | ❌ Missing |
| Referrer-Policy | ❌ Missing |
| Permissions-Policy | ❌ Missing |
| Cross-Origin-Opener-Policy | ❌ Missing |

### Risks (Before)

1. **No CSP** — Any injected script (XSS) executes freely. No restrictions on external resources.
2. **No HSTS** — First visit (or MITM on initial redirect) can be downgraded to HTTP.
3. **No XFO** — Application is embeddable in third-party iframes (clickjacking).
4. **No X-CTO** — Browsers may MIME-sniff resources, converting e.g. user-uploaded text into executable script.
5. **No Referrer-Policy** — Full URL (including path/query) leaked to every external link destination.
6. **No Permissions-Policy** — All browser APIs (camera, mic, geolocation) available to third-party scripts.
7. **No COOP** — Cross-origin windows retain full `opener` reference.

## Implemented Headers

Defined in `next.config.ts:71-78` via the `headers()` callback.

| Header | Value | Reason |
|---|---|---|
| Content-Security-Policy | See CSP section below | Restricts resource loading to trusted origins |
| Strict-Transport-Security | `max-age=63072000; includeSubDomains; preload` | 2-year HTTPS enforcement, subdomain coverage, preload eligibility |
| X-Frame-Options | `DENY` | Prevents clickjacking by blocking all iframe embedding |
| X-Content-Type-Options | `nosniff` | Forces browsers to honor declared Content-Type |
| Referrer-Policy | `strict-origin-when-cross-origin` | Full referrer on same-origin; origin-only cross-origin; dropped on HTTPS→HTTP |
| Permissions-Policy | `camera=(), microphone=(), geolocation=(), interest-cohort=()` | Disables unused APIs. `interest-cohort=()` opts out of FLoC/PCM |
| Cross-Origin-Opener-Policy | `same-origin` | Isolates window references; prevents cross-origin leaks |

### Headers Intentionally Omitted

| Header | Reason Omitted |
|---|---|
| Cross-Origin-Embedder-Policy | Would require `Cross-Origin-Resource-Policy: cross-origin` on all cross-origin resources (Cloudinary images, Google Fonts). This project does not use `SharedArrayBuffer` or need cross-origin isolation. Setting it would break CDN-loaded resources. |
| Cross-Origin-Resource-Policy | Would block social media crawlers (Twitter, Facebook, Discord) from fetching OG images (`/preview.png`, `/me.jpg`) and profile picture. Those crawlers fetch as cross-origin `no-cors` requests and would receive empty responses. |

## Content Security Policy

### Full Policy

```
default-src 'self';
script-src 'self' 'unsafe-inline';
style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
img-src 'self' https://res.cloudinary.com;
font-src 'self' https://fonts.gstatic.com;
connect-src 'self' https://vitals.vercel-analytics.com;
frame-src 'none';
frame-ancestors 'none';
base-uri 'none';
form-action 'self'
```

### Directive Breakdown

#### `default-src 'self'`
Base restriction. All resource types fall back to same-origin unless overridden.

#### `script-src 'self' 'unsafe-inline'`
**Exception: `'unsafe-inline'`**

Required by Next.js App Router for:
- Inline bootstrap scripts that initialize the React runtime
- RSC payload scripts containing page data
- Client-side navigation scripts
- Dev overlay scripts (development only)

These inline scripts are generated by Next.js framework code and are not user-controllable. No user code or third-party library requires `'unsafe-inline'`.

Not required and NOT present: `'unsafe-eval'`, `'strict-dynamic'`, `'nonce-*'`. GSAP 3.15, Lenis, and all other dependencies work without eval. Hash-based CSP is impractical here because Next.js inline script content varies between builds.

#### `style-src 'self' 'unsafe-inline' https://fonts.googleapis.com`
**Exceptions: `'unsafe-inline'`, `https://fonts.googleapis.com`**

- `'unsafe-inline'` — Required by Next.js critical CSS injection and by React inline `style` props used in `layout.tsx` (noscript fallback, Toaster theming) and GSAP-driven components that apply inline transforms. CSS-in-JS is not used; all Tailwind styles are compiled to static files.
- `https://fonts.googleapis.com` — Required for the Material Symbols Outlined stylesheet loaded via `<link>` in `layout.tsx:129-132`.

#### `img-src 'self' https://res.cloudinary.com`
`data:` and `blob:` explicitly excluded. No component in this project uses data URIs for images. Adding them would allow embedding arbitrary content without HTTP round-trips. If a future feature needs base64 images, this should be reconsidered with specific justification.

#### `font-src 'self' https://fonts.gstatic.com`
Geist, Inter, and JetBrains Mono are self-hosted via `next/font/google` (built into the application bundle). Material Symbols Outlined font files are served from `fonts.gstatic.com`, referenced by the Google Fonts stylesheet.

#### `connect-src 'self' https://vitals.vercel-analytics.com`
- `'self'` — Covers Server Action POST requests (contact form submission, project fetching).
- `https://vitals.vercel-analytics.com` — Required by `@vercel/analytics` and `@vercel/speed-insights` for telemetry data submission. The endpoint uses `navigator.sendBeacon` and `fetch`.

#### `frame-src 'none'`, `frame-ancestors 'none'`
No content in this application is designed for iframe embedding. Blocking frames prevents clickjacking and UI redress attacks.

#### `base-uri 'none'`
Prevents attackers from injecting `<base>` tags to hijack relative URLs.

#### `form-action 'self'`
Restricts form submissions to same-origin only. The contact form uses a Server Action (POST to self).

### Origins Not Included (by design)

| Origin | Reason Excluded |
|---|---|
| `https://github.com` | Navigation links only (user-initiated clicks). Scripts/styles/fonts must not load from GitHub. Users clicking external links navigate away from the app. |
| `https://linkedin.com` | Same as above |
| `https://discord.com` | Same as above |
| `https://drive.google.com` | CV link — user-initiated navigation only |
| `https://ymdev.me` | Already covered by `'self'` |
| `https://web.facebook.com` | Linked in JSON-LD `sameAs` only |

## Implementation Location

### Chosen: `next.config.ts` `headers()` function

**Rationale:**

1. **Framework-level** — Next.js applies these headers during its response lifecycle, before Vercel's edge network processes them. This is the most reliable layer for security headers.
2. **Static per-route patterns** — Since the policy is the same for all routes (no dynamic nonce needed), `next.config.ts` is ideal. No per-request computation is required.
3. **Vercel compatibility** — Vercel's documentation explicitly recommends `next.config.ts` for security headers. Headers set here are respected by Vercel's edge cache and do not risk being stripped.

**Why not middleware?**

- Middleware headers can be stripped by CDN caching layers on Vercel.
- Middleware runs on every request (edge compute cost, latency).
- No dynamic nonce is needed (this project uses `'unsafe-inline'` with justification), so middleware's main advantage for CSP doesn't apply.

**Why not vercel.json?**

- No `vercel.json` exists in this project. Adding one would split configuration across files. Framework-level config is preferred.

## Future Recommendations

1. **CSP Reporting** — Add a `report-uri` or `report-to` directive pointing to a reporting endpoint (e.g., `https://ymdev.report-uri.com/r/d/csp/enforce`). This would catch any undiscovered violations in production.

2. **Nonce-based CSP** — If stronger isolation is desired, migrate to a nonce-based CSP:
   - Create `middleware.ts` that generates a cryptographic nonce per request
   - Set `x-nonce` response header (Next.js picks this up for inline scripts)
   - Replace `'unsafe-inline'` with `'nonce-<value>'`
   - This removes both `'unsafe-inline'` exceptions but adds middleware complexity and edge compute cost per request.

3. **Subresource Integrity** — Add SRI hashes to the Material Symbols `<link>` stylesheet in `layout.tsx` to prevent CDN compromise:
   ```html
   <link rel="stylesheet"
         href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined:opsz,wght,FILL,GRAD@24,400,0,0&display=swap"
         integrity="sha384-..."
         crossorigin="anonymous" />
   ```

4. **Remove unused dependencies** — Consider whether `mammoth` (docx parser) and `puppeteer-core` are needed in production. If not, move to `devDependencies` to reduce bundle size and attack surface.

5. **Security.txt** — Add `public/.well-known/security.txt` with a disclosure policy for responsible reporting.

6. **CSP Report-Only mode** — Before deploying major changes, consider adding `Content-Security-Policy-Report-Only` alongside the enforced policy to catch regressions before they block users.
